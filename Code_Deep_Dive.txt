CODE DEEP DIVE: ARCHITECTURE & PATTERNS

This document explains EXACTLY how the core features of the Neural-AI Resume Analyzer are built so you can maintain them, expand them, and copy the patterns for future features.

=========================================
1. THE GLOBAL AUTH & BACKEND STORE (ZUSTAND)
=========================================
File: `app/lib/puter.ts`

We use Zustand to create a globally accessible "Store". This is much cleaner than passing props down through 10 levels of components or setting up complex React Context providers.

Why it's structured this way:
* Puter.js is injected via a `<script>` tag in `root.tsx`.
* It takes a moment to load from the internet.
* The `init()` function runs a `setInterval` that checks 10 times a second to see if `window.puter` finally exists.
* Once it exists, it clears the interval and checks if the user is already logged in.

How to use it in any component:
```javascript
import { usePuterStore } from "~/lib/puter";

export default function MyPage() {
    // 1. Grab exactly the functions or states you need
    const { auth, isLoading, puterReady } = usePuterStore();

    // 2. Safely check state
    if (isLoading || !puterReady) return <p>Loading SDK...</p>;

    // 3. Trigger actions directly
    return (
        <button onClick={auth.signIn}>
            {auth.isAuthenticated ? "Switch Account" : "Log In"}
        </button>
    )
}
```

2. THE "SMART FORM" PATTERN
=========================================
File: `app/routes/upload.tsx`

This form is built as a "Controlled Component". This means React state (`useState`) is the single source of truth for every input, rather than the HTML DOM.

Why this is powerful:
Because we track every keystroke in state, we can write complex real-time logic that updates the UI instantly.

Example: The "Clear Text" Button Logic
```javascript
const [jobTitle, setJobTitle] = useState("");
const [companyName, setCompanyName] = useState("");

// This variable recalculates on every render.
// It is TRUE if ANY of the fields have text. It is FALSE if they are all empty.
const hasTextData = jobTitle.trim() !== "" || companyName.trim() !== "";

// In the JSX, we only render the button if hasTextData is true!
{hasTextData && (
    <button onClick={handleClearForm}>Clear</button>
)}
```

3. THE DRAG-AND-DROP UPLOADER
=========================================
File: `app/components/FileUploader.tsx`

We use the `react-dropzone` library instead of writing manual HTML5 drag events (`onDragEnter`, `onDrop`).

Why?
1. It handles cross-browser bugs automatically.
2. It gives us an easy `accept` object to restrict to PDFs/Word docs.
3. It handles `maxSize` rejection gracefully.

How the data flows back to the parent:
```javascript
// Inside FileUploader.tsx:
const onDrop = useCallback((acceptedFiles) => {
    const file = acceptedFiles[0]; // Grab the first valid file
    setSelectedFile(file);         // Update local UI state (to show the checkmark)
    onFileSelect(file);            // Pass the data UP to the parent component!
}, [onFileSelect]);

// Inside upload.tsx (the parent):
<FileUploader onFileSelect={(file) => setFile(file)} />
```

4. ORCHESTRATING FRAMER MOTION ANIMATIONS
=========================================
File: `app/components/ResumeAnalyzerAnimation.tsx`

React components usually instantly snap onto the screen. Framer Motion lets us build complex, timeline-based keyframe animations.

The Keyframe Array Pattern:
Instead of saying `x: 100`, you can give it an array: `x: [0, 80, 0]`. Framer Motion will smoothly transition the element through those points over the given `duration`.

The Timeline Pattern (`times`):
```javascript
animate={{
    scale:   [1, 1.3, 0.2], // Normal -> Big -> Tiny
    opacity: [0, 1,   1, 0] // Invisible -> Visible -> Visible -> Invisible
}}
transition={{
    duration: 1.5,
    // These numbers dictate WHEN the keyframes above happen (0 to 1)
    // 0 = Start, 0.4 = 40% through the animation, 1 = End
    times: [0, 0.4, 0.9, 1]
}}
```

The Callback Pattern:
Because the animation is a separate component, the parent (`upload.tsx`) doesn't know when the animation finishes.
1. The parent passes an `onAnimationComplete` function down.
2. The animation component runs a `setTimeout` matching the exact length of the animation (e.g., 2.5s).
3. When the timeout fires, it calls the prop function.
4. The parent receives the signal, hides the animation, and shows the loading screen!
